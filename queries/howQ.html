How CFRMIS Queries Work

Operators

CFRMIS offers two types of search function to give you access to the database. One is a General Search function similar to what you may be used to from other applications. The other, Queries, is a more specific interrogation of the database entries.

To effectively search the CFRMIS database you will need to use search operators. These expand or limit the scope of a search to ensure that you find the correct record(s).

To illustrate, imagine that we were trying to find the Premises record for Adam’s Fancy House of Cheeses. We might try typing the below into the Occupier field of a premises query:

House of Cheese

But this wouldn’t give us the result that we wanted. This will only find premises where the Occupier value is exactly “House of Cheese”.

Note

This isn’t strictly true. CFRMIS queries don’t take into account capitalisation, so this search term could also return “house of cheese”, or “hOuSe OF cHeESe” for that matter.

If we want to expand our search term to find Premises with an Occupier containing House of Cheese we would instead type the following:

%House of Cheese%

This would return Adam’s Fancy House of Cheeses, as well as Karen’s Discount House of Cheese and Biscuits and any of the many other places with House of Cheese in their name.

Due to the inconsistencies in the way information has historically been entered onto CFRMIS it is essential that you use the search operators to find the correct Premises records and ensure that we are collating data that belong together.

CFRMIS Search Operators

Operator

Description

%/*

This is a “wildcard”. These operators both do the same thing and can be used interchangeably. They are used to represents an arbitrary number of characters, even zero characters.

_

The underscore is similar to the above wildcard operators but only represents a single character. Multiple underscores can be used together if you have a defined number of arbitrary characters in mind.

![term]

The exclamation mark means “not”, so ![term] means “not [term]”. This means that the search will return everything except “[term]”.

&

The ampersand is a “null” operator. This means that the search will return all records where this field is empty.

Examples

a%

will return any records where this value starts with “a”.

*b

will return any records where this value ends with “b”.

%cd%

will return any records where this value contains “cd” anywhere.

_e*

will return any records where this value has an “e” as the second character.

f__%

will return any records where this value starts with “f” and is at least three characters long.

g*h

will return any records where this value begins with “g” and ends with “h”.

!i

will return any records where this value is not “i”.

&

will return any records where this value is empty.

Form Queries

For every type of record in CFRMIS there is a form used to interact with the information in those records. One of these interactions is querying the database to find records that match one or more criteria.

You can make a query using any of the information which is recorded as part of a record. Every empty field that you see in the Query form can be used as part of your query. If you put some information into a field CFRMIS will return every record which matches what you put into that field.

Note

The information that you put into a field is a term - a Premises Record will only appear in the results of a query if the information that it has in the respective field matches the term that you have entered.

This also works with no terms at all.

Press Other Buildings to open a Query Other Building form.

Press Search. A results pane will open at the bottom of the screen displaying every single Premises Record which is not a domestic premises.

We’ll look at the results pane in a little more detail soon, but for now just look at the bottom-right-hand corner of the pane. This search has given us over 20 000 results. This list has its uses but for most activites we need to trim it down to be a little more specific.
