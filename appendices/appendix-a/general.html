General

General expectations for entry of premises records.

Main Details

Occupier

This is, as it sounds, the occupier, the primary business or organisation operating from the premises.

Do:

  • Keep it simple - style the premises in the same way as the occupier does.

  • Make sure that the spelling is correct

Don’t:

  • Add unnecessary or presumed information to the occupier’s name, like corporate structure acronyms (Ltd., Plc.) unless this is explicitly how the occupier brands itself and is commonly known.

  • Use this field to record any additional information, like previous occupiers.

Note

“KFC” not “Kentucky Fried Chicken”

“Sports Direct” not “SportsDirect”, but “SportsWorld” not “Sports World”

“Frankie & Benny’s” not “Frankie and Bennys”

“B&Q” not “B and Q plc.”

“Elysium” not “Elysium (ex-Aura)”

Property No.

The number of the property, where it has one. If the property is not numbered then this field should be left blank.

Note

Remember that although it is important to fill out all of our records as thoroughly as possible it is even more important to make sure that we only enter information that we are sure is true - don’t make any guesses! This is how we end up with a an inaccurate, messy database!

Address Lines

These should ideally be filled out using the gazetteer in order to ensure consistency across the database.

Postcode

Remember, if you have used the Using Edit Map method with a 7-character post code to make sure that you put the space back into this field - “NN14 1UE” instead of “NN141UE”

FS Team

The only two options that shoould be used here are FP West, for premises in West Northamptonshire, and FP North, for premises in North Northamptonshire. The other codes are historic and should not be used. Where these are encountered as part of existing premises records they should be changed to reflect the appropriate team.

Parish

This field should be used to record the parish in which the premises is situated. The code lookup button is a must here - there are 305 parishes across the county. Make sure you make use of the filter to type in the name of the parish you’re looking for.

Sup Line No.

The supplementary line number is a code containing information relating to the premises type. As there are a number of recording and reporting systems in place, both historical and current, there are several different code schemes that are recognised by CFRMIS. Rather than choosing each of these individually selecting a supplementary line number will automatically populate the below fields appropriately.

The supplementary line number codes are largely based on the CLG Line No. discussed below, with some local additions. It is important that you select the correct supplementary line number as this determines all sorts of behind-the-curtain information about the premises. See Supplementary Line Number.

Valuation Office

The code used by the Valuation Office Agency for tax-related assessments. This field will be automatically populated when you select a supplementary line number.

CLG Line No.

The Community and Local Government Line Number is a little tricky to work out the provenance of. When the Fire Safety Order was introduced the Department for Communities and Local Government (DCLG) provided much of the supporting guidance documents which are still used to this day. The Department later became a Ministry for a while and is now the Department for Levelling Up, Housing and Communities. The supplementary line number codes that we use are based on these CLG codes, though any modern reference to them is difficult to find. This field will be automatically populated when you select a supplementary line number.

FSEC Code

The Fire Safety Emergency Cover classification comes from a software toolkit used by fire and rescue services to assess risk in their area. The categories it covers are very similar to those used by the Home Office for the collation of national fire safety statistics. This field will be automatically populated when you select a supplementary line number.

TOID

The Topographical Identifier is a 13-16 digit long number which provides a unique reference for some 440,000,000 geographical features across Britain. It is used by the Ordnance Survey AddressBase gazetteer that we now use. This field will be automatically populated when you copy address details from the gazetteer.

NLPG-UPRN

The National Land and Property Gazetteer Unique Property Reference Number. This UPRN is a unique identifier for every addressable location in the country.

OSAPR

The Ordnance Survey Address-Point Reference is an identifier used by the Ordnance Survey Address-Point gazetteer which was formerly in use within NFRS. Some existing premises will have an entry in this field, but this is a historical feature and you do not need to enter anything into this field when creating a new premises.

Gazetteer ID

This field is not used within NFRS.

Demographic Code

This field is not used within NFRS.

Dwelling Type

This field is primarily used with Domestic Premises rather than Other Buildings, however it may be appropriate to use this where it is necessary to record Fire Protection activity that takes place within a Domestic Premises.

Note

Remember that a HMO is not a dwelling, it is a collection of dwellings (i.e. the bedrooms) and that a HMO premises in Other Buildings is the common the parts of the premises which are subject to the Fire Safety Order.

Risk Level

This field contains the risk level recorded on the most recent TFS job against the premises. When creating a Premises record this will typically be set to Medium.

Compliance Level

This field contains the compliance level recorded on the most recent TFS job against the premises. When creating a Premises record this will typically be set to Average.

Inspection Frequency

This field is automatically populated by CFRMIS based on the premises type (supplentary line number), risk level and complaince level. You do not need to enter anything into this field when creating a new premises.

No. of Persons

This is used to record the maximum occupancy of the premises. Although you may record this for any premises type (where you know the correct figure, no guesses! See Data Integrity) it is particularly important for certain premises types, and is discussed further in the appropriate sections of Appendix A - Protection Guidance Notes.

Multiple Occupancy?

Is this premises one of multiple premises occupying a building? The value entered in this field must be “Y” for Yes or “N” for No.

Other Legislation?

Is there other legislation regulating fire safety in this premises? The value entered in this field must be “Y” for Yes or “N” for No.
